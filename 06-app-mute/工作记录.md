# AppMute v06 工作记录

## 项目状态

**可运行**，发布版在 `bin\Release\net8.0\win-x64\publish\AppMute.exe`（12MB 单文件，无需安装 .NET）。

## 做了什么

从 AppMute 归档的三个旧版本（Python/C#/C）重新用 C# .NET 8 构建了一个版本。旧版本全都有致命 bug 导致静音功能不工作。

## 旧版本的问题

| 版本 | 问题 |
|------|------|
| 03 (Python) | 唯一能工作的版本，但依赖重（pycaw/pystray/keyboard） |
| 04 (C#) | COM GUID 全写错了；vtable 截断（只声明 1 个方法，实际 vtable 有 10+ 个）；COM 初始化在构造函数线程，WndProc 在另一个线程，CoCreateInstance 返回 E_NOINTERFACE |
| 05 (C) | IID_ISimpleAudioVolume 拷成了 IID_IAudioSessionManager2 的值；无托盘图标；无配置热重载 |

## 新版本 v06 的架构

单文件 `Program.cs`，约 490 行，按功能分区：

- **COM 接口定义**（行 22-111）— 完整 vtable 布局，GUID 从 Windows SDK 头文件确认
- **Win32 P/Invoke**（行 113-186）— RegisterHotKey、Shell_NotifyIconW、消息循环等
- **主程序**（行 188-末尾）— 窗口、托盘、热键、音频控制、配置管理

核心流程：`主线程 COINIT_APARTMENTTHREADED` → `创建隐藏窗口` → `RegisterHotKey` → `Shell_NotifyIconW` → `消息循环` → `WM_HOTKEY 时 ToggleMute`

## 踩过的坑（重要）

### 1. CreateWindowExW 必须加 CharSet.Unicode

`CreateWindowExW` 的 DllImport 如果不加 `CharSet = CharSet.Unicode`，字符串参数按 ANSI 编码传给 Unicode 版函数，窗口类名匹配不上，CreateWindowExW 返回 0。表现：RegisterHotKey 用 hwnd=0 注册（不报错但无效），Shell_NotifyIconW 失败。

```csharp
// 错误
[DllImport("user32.dll")] public static extern IntPtr CreateWindowExW(...);

// 正确
[DllImport("user32.dll", CharSet = CharSet.Unicode, SetLastError = true)]
public static extern IntPtr CreateWindowExW(...);
```

### 2. .NET 8 Trimmed 发布默认禁用 COM

`PublishTrimmed=true` 时，.NET 8 默认关闭 `BuiltInComInteropSupport`，所有 `[ComImport]` 接口在运行时抛异常："Built-in COM has been disabled via a feature switch"。csproj 必须显式启用：

```xml
<BuiltInComInteropSupport>true</BuiltInComInteropSupport>
```

### 3. WNDCLASSW 的 lpfnWndProc 不能用委托类型

`WNDCLASSW.lpfnWndProc` 字段声明为 `WndProcDelegate` 类型时，Marshal 给 RegisterClassW 传的函数指针可能不对。改为 `IntPtr` + `Marshal.GetFunctionPointerForDelegate`：

```csharp
struct WNDCLASSW {
    public IntPtr lpfnWndProc; // 不是 WndProcDelegate
    ...
}
// 使用时：
wc.lpfnWndProc = Marshal.GetFunctionPointerForDelegate(_wndProc);
```

`_wndProc` 必须是 static 字段，防止 GC 回收委托导致函数指针悬空。

### 4. Shell_NotifyIconW 在 shell32.dll 不在 user32.dll

```csharp
[DllImport("shell32.dll")] // 不是 user32.dll
```

### 5. GetLastError() 在 .NET 中不可靠

P/Invoke 调用之间 CLR 可能调用其他 Win32 函数，覆盖 last error。必须用 `SetLastError=true` + `Marshal.GetLastWin32Error()`。

### 6. Process.Start 打开文件

.NET 8 默认 `UseShellExecute=false`，`Process.Start("notepad.exe", path)` 可能失败。改用：

```csharp
Process.Start(new ProcessStartInfo(path) { UseShellExecute = true });
```

直接用系统关联程序打开文件，不硬编码 notepad。

### 7. COM GUID 来源

GUID 不能从网上搜（搜索结果互相矛盾且经常编造）。从以下来源确认：
- NAudio 源码（GitHub naudio/NAudio）
- Windows SDK 头文件（GitHub tpn/winsdk-10 仓库的 audiopolicy.h / mmdeviceapi.h）
